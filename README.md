# OPC Industry KB

面向 OPC / 一人公司 / 超级个体 的垂直行业 AI 知识库项目。

## 项目说明
这个项目不是在做一个普通“知识库”，而是在做一个 **可被 AI 调用的行业生产系统**。

在 OPC 场景里，真正的壁垒不是“会用 AI”，而是：
- 是否有自己的行业知识资产
- 是否有结构化模板和 SOP
- 是否能把经验沉淀为可复用工作流
- 是否能通过反馈不断优化结果

所以，这个仓库的目标是把某个垂直行业的：
- 术语
- 规则
- 模板
- 案例
- 工作流
- 反馈数据

组织成一套可以持续积累、持续调用、持续优化的 AI 基础设施。

---

## 适用对象
- 独立招聘顾问 / 独立顾问 / 小团队主理人
- 想把 AI 从“偶尔用一下”变成“稳定生产工具”的 OPC
- 想做垂直行业 AI 知识库 / AI 副驾 / AI 工作流产品的人

---

## 项目目标
不是做一个“资料堆”，而是做一个可复用的行业 AI 生产系统：
- 能查
- 能生成
- 能沉淀反馈
- 能持续优化

一句话说：

> 把行业经验，变成 AI 可以稳定调用的生产能力。

---

## 当前结构
- `docs/`：方法论、schema、路线图
- `knowledge/`：行业知识资产
- `templates/`：可复用模板
- `workflows/`：任务工作流
- `feedback/`：反馈回流规范
- `schemas/`：结构化数据 schema
- `data/`：原始数据、清洗数据、向量化中间产物
- `verticals/`：具体垂直行业版本

---

## 核心原则
1. 先选垂直行业，再建知识库
2. 先打一个高频任务，不要大而全
3. 向量库不是壁垒，行业资产 + 工作流 + 反馈闭环才是
4. 用户买的不是“RAG”，而是更快、更稳、更省人力的结果
