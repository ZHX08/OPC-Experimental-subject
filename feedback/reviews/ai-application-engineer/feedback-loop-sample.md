# AI 应用工程师反馈回流样例

> 这个样例展示一轮最小闭环：JD + 简历 -> 匹配评估 -> 人工修改 -> 最终推荐 -> 回流记录。

## 1. 任务背景
- 任务类型：AI 应用工程师候选人匹配评估
- 目标岗位：AI 应用工程师
- 输入来源：JD 样本 + 候选人简历样本
- 目的：判断是否进入推荐报告阶段

## 2. AI 初稿摘要
- 推荐级别：可推
- 理由：候选人具备 RAG、Prompt、workflow 和基础工程能力，做过企业知识助手与销售方案生成助手。
- 主要判断：技术栈和项目类型较匹配，但证据链不够完整。

## 3. 人工修改点
- 把“会做 RAG”改成“能说明 chunking / rerank / 命中率 / 反馈回流”
- 补充“是否上线、多少用户在用、是否稳定运行”
- 补充“成本、延迟、fallback、人工确认点”
- 去掉过于空泛的“技术全面”表述，改为可验证证据

## 4. 最终输出摘要
- 推荐级别：可推
- 原因：候选人能做 AI 应用工程落地，且有较完整的评测与回流意识；需要在终面继续确认线上规模和成本控制细节。

## 6. 可回流的规则
- 如果候选人没有上线证据，推荐等级自动降一级
- 如果没有评测和反馈记录，报告必须标注“证据不足”
- 如果无法说明 fallback / 兜底方案，进入终面前必须补问
