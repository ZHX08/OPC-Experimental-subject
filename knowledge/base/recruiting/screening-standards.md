# 招聘初筛标准（第一版）

## 通用初筛逻辑
1. 先看岗位匹配度
2. 再看项目复杂度
3. 再看职责边界
4. 再看结果产出
5. 最后看风险项

## 通用必须问
- 你在这个项目里具体负责什么？
- 最终结果是什么？
- 遇到的最大问题是什么？
- 你是怎么推进解决的？

## 通用风险信号
- 简历堆砌关键词，但没有具体项目
- 说不清个人贡献
- 频繁跳槽但没有合理解释
- 只有执行经历，没有判断和结果

---

## 前端工程师
### 优先看
- 框架熟练度
- 模块独立负责经历
- 工程化能力
- 线上问题处理能力

### 高风险
- 只有外包切图经验
- 缺少真实业务项目
- 基础概念模糊

---

## 产品经理
### 优先看
- 是否主导需求闭环
- 是否有数据结果
- 是否具备业务理解与优先级判断

### 高风险
- 文档型产品
- 无法量化结果
- 推进能力弱

---

## AI 应用工程师
### 优先看
- 是否有可展示项目
- 是否做过工作流 / 知识库 / Agent
- 是否理解效果优化与异常处理

### 高风险
- 只会调用 API
- 只有 demo，没有上线经验
- 只讲模型，不讲业务

---

## 增长运营
### 优先看
- 是否有明确增长结果
- 是否理解渠道到转化闭环
- 是否有试错和复盘能力

### 高风险
- 只会发内容
- 没有数据意识
- 不能解释增长动作与结果关系
