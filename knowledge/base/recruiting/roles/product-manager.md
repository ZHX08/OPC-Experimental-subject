# 产品经理岗位画像

## 岗位目标
围绕业务目标完成需求定义、方案设计、协作推进和结果复盘，推动产品持续迭代。

## 核心职责
- 负责需求调研、优先级判断、PRD 输出
- 协调设计、研发、测试、运营推进上线
- 跟踪关键指标并持续优化
- 管理版本节奏与跨团队协作

## 必须项
- 具备完整需求闭环经验：发现问题 → 定义方案 → 推动上线 → 结果复盘
- 能写清楚 PRD、流程图、原型
- 能说明指标意识和优先级判断逻辑
- 有跨团队推进能力

## 常见加分项
- 有 AI / 数据产品 / B 端系统经验
- 有增长、商业化、平台化经验
- 有 0→1 产品孵化经验
- 会基础 SQL / 数据分析

## 常见风险项
- 只会写文档，不会推进落地
- 只讲功能，不讲业务目标
- 项目里职责边界不清，实际是助理型角色
- 缺少结果数据与复盘能力

## 面试重点
- 做过最复杂的一个需求是什么，为什么这样设计
- 如何做优先级取舍
- 指标怎么定，结果怎么验证
- 与研发/业务有冲突时怎么推进

## 初筛淘汰标准
- 只描述参与过程，无法说明自己主导部分
- 没有可量化结果
- 不理解 B 端 / C 端差异
- 逻辑表达混乱，需求抽象能力弱
