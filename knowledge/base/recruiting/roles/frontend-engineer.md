# 前端工程师岗位画像

## 岗位目标
负责 Web 前端页面与交互实现，推动业务需求快速上线，并保证性能、可维护性与协作效率。

## 核心职责
- 负责 PC / H5 / 管理后台前端开发
- 与产品、设计、后端协作完成需求上线
- 维护组件、工程化、发布流程
- 持续优化性能、稳定性、可维护性

## 必须项
- 熟练掌握 JavaScript / TypeScript
- 熟练掌握 React 或 Vue 其一
- 理解组件化、状态管理、路由、接口联调
- 具备基本工程化能力：Vite / Webpack / npm / pnpm / Git
- 能独立完成中等复杂度页面和模块开发

## 常见加分项
- 有复杂后台、低代码、可视化、编辑器经验
- 有性能优化、监控、埋点经验
- 有 Node.js / SSR / Next.js / Electron 经验
- 有 AI 产品前端经验，如 Agent、聊天 UI、流式输出、知识库前台

## 常见风险项
- 只会切图，不理解业务与状态管理
- 只会写页面，不会定位线上问题
- 项目经验偏简单，缺少独立负责模块经历
- 简历堆砌技术名词，但缺少真实产出

## 面试重点
- 最近一个项目中你独立负责了什么
- 如何拆复杂页面与组件
- 遇到线上性能 / Bug / 接口问题怎么处理
- 如何与产品和后端协作推动上线

## 初筛淘汰标准
- 没有主流框架项目经验
- 只写静态页面，缺少真实业务项目
- 无法说明自己负责边界
- 基础概念混乱：闭包、异步、组件通信、状态管理
