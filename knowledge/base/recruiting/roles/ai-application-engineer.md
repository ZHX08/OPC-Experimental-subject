# AI 应用工程师岗位画像

## 岗位目标
基于大模型、工作流、知识库、Agent 等能力，快速搭建可落地的 AI 应用，并持续优化效果与稳定性。

## 核心职责
- 搭建 AI 应用原型与生产版本
- 处理 Prompt、RAG、工具调用、工作流编排
- 设计评测、反馈与优化机制
- 与业务方协作完成场景落地

## 必须项
- 有 LLM 应用实战经验，不是只会调用 API
- 理解 Prompt、上下文、工具调用、知识库检索基本原理
- 具备 Python / Node.js 至少一种工程能力
- 能把业务问题拆成可执行流程
- 有调试效果、控制成本、处理异常的经验

## 常见加分项
- 有 Dify / n8n / LangGraph / OpenAI SDK / MCP 等实战
- 有评测、日志、反馈回流、数据清洗经验
- 有 AI Coding、企业知识库、客服、销售场景经验
- 理解 embedding、rerank、chunking，但不迷信技术堆砌

## 常见风险项
- 只会做 demo，不会做稳定系统
- 过度依赖单一模型能力，没有 fallback 思维
- 只讲技术术语，不讲业务结果
- 不理解成本、效果、延迟之间的平衡

## 面试重点
- 做过的 AI 应用解决了什么业务问题
- 知识库 / RAG 是怎么设计的
- 效果不稳定时如何排查
- 如何做 prompt、工作流、反馈闭环优化

## 初筛淘汰标准
- 没有真实上线或可演示项目
- 只能讲模型，不会讲任务流程
- 不理解 hallucination、评测、成本控制
- 无法说明自己做了哪些关键工程工作
