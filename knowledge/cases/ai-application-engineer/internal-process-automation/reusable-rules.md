# Reusable Rules: 内部流程自动化

这些规则可以复用到招聘筛选、推荐报告、销售样板和复盘沉淀里。

## R-AIE-IFW-01
- 触发条件：场景里出现 ERP / OA / 审批 / 工单 / 合同 / 采购 / 法务
- 判断规则：必须补 system of record 和回写路径
- 输出动作：没有就标记为 `needs_follow_up`

## R-AIE-IFW-02
- 触发条件：候选人只讲“会 workflow”“会 prompt”“会摘要生成”
- 判断规则：如果说不出入口、回写、审计和 fallback，不能算内部流程自动化能力
- 输出动作：降级为 `cautious_recommend` 或直接打回

## R-AIE-IFW-03
- 触发条件：存在高风险字段，例如金额、供应商、法律条款、审批结论
- 判断规则：必须有人工确认点
- 输出动作：没有人工确认，就不能进入强推荐

## R-AIE-IFW-04
- 触发条件：方案说了 fallback
- 判断规则：必须同时说清楚触发条件、回退对象和责任人
- 输出动作：只说“人工兜底”不算完整

## R-AIE-IFW-05
- 触发条件：没有上线证据或试点数据
- 判断规则：不能把 ROI 当结论，只能当假设
- 输出动作：标注证据不足，暂不建议深挖

## R-AIE-IFW-06
- 触发条件：出现“自动提交”或“自动通过”
- 判断规则：要检查审计、回滚和监控是否齐全
- 输出动作：缺任一项，默认降级

## R-AIE-IFW-07
- 触发条件：要卖“内部流程自动化”样板
- 判断规则：必须同时给出成功案例和失败对照
- 输出动作：只有成功案例不算完整样板

## R-AIE-IFW-08
- 触发条件：需要写推荐报告或销售一页纸
- 判断规则：必须落到“业务场景 / 工程链路 / fallback / 审计 / ROI / 风险边界”六件事
- 输出动作：缺任何一件，都不算成熟包
