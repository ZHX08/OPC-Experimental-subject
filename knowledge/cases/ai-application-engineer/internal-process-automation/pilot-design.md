# AI 应用工程师 - 内部流程自动化真实试点设计

这是 `internal-process-automation` 方向的知识侧摘要，和售卖侧设计保持一致。

## 试点场景
- 场景：采购申请预审与分流
- 模式：shadow mode 起步
- 目标：先跑通单流程、单入口、单 SoR、单人审点、单回写路径

## system of record
- OA 审批系统：最终审批状态与流转真源头
- ERP 供应商主数据：供应商和预算科目参考源
- 工单系统：异常升级与退回记录

## 人审点
- 字段完整性确认
- 高风险字段确认
- 最终提交前复核

## 回写路径
- 本地 UI 录入试点输入
- 生成运行评估和人工确认
- 保存试点证据
- 输出反馈回写包到 `feedback/reviews/ai-application-engineer/`
- 后续整理进 SQLite 方向库

## 失败兜底
- 缺字段：退回补充
- 低置信度：只出草稿
- 高风险字段：必须人工确认
- 接口异常：保留草稿和审计记录

## 这条线为什么值得做
- 规则边界清楚
- 回写路径清楚
- 证据好留
- ROI 容易算
- 可以很快判断是否值得继续加码
