# Rule Card: 内部流程自动化的硬门槛

这张卡只管一件事：**这个候选人 / 这个方案，能不能被当成“可上线、可审计、可回滚”的内部流程自动化能力**。

## 硬门槛

1. **必须有具体业务场景**
   - 至少说清楚是采购、法务、审批、工单还是运营流转
   - 不能只写“做工作流”

2. **必须说清系统链路**
   - 入口、处理、回写、人工确认、异常处理要完整
   - 不能只有“生成一段摘要”

3. **必须有审计点**
   - 输入、模型版本、人工编辑、最终结果都要可追踪
   - 没审计，默认不稳定

4. **必须有 fallback**
   - 至少有规则模板回退、人工确认或草稿保留
   - 没 fallback，默认不能卖

5. **必须有评测 / 反馈闭环**
   - 要能说明如何验证结果可用
   - 要能说明 human_edits 怎么回写

6. **必须有部署、监控和回滚意识**
   - 没有监控和回滚，只能算 demo 能力

7. **必须有成本 / 延迟边界**
   - 如果无法说明时延和调用成本，默认只能谨慎推荐

## 打分规则

- 上面 7 条全部满足：可以进入强推荐或可卖样板
- 缺 1-2 条：降级为谨慎推荐
- 缺业务场景 / 审计 / fallback / 回滚 任一项：直接打回

## 典型追问

- 这条流程的系统 of record 是什么？
- 哪个步骤会人工确认？为什么？
- 模型失败时具体回退到哪里？
- 监控什么指标？谁来处理异常？
- ROI 是怎么量化的？
