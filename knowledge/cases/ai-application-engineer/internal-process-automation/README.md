# AI 应用工程师 - 内部流程自动化

这组内容只讲 **AI 应用工程师 -> 内部流程自动化**，不横向扩到客服、销售或知识工作台。

## 为什么单独拆出来

内部流程自动化是最容易把“会做 demo”误判成“能上线”的方向。
它真正需要看的不是一句“会 workflow”，而是下面这些硬证据：

- 业务场景是否具体，是否能说清系统链路
- 是否接了系统 of record（ERP / OA / 审批 / 工单）
- 是否有评测、日志、审计、回滚、监控
- 是否有 fallback 和人工确认点
- 是否能量化 ROI、延迟和成本边界

## 目录内容

- `case-card-success.md`：一个可卖的成功样例
- `case-card-failure.md`：一个不能卖的失败对照
- `rule-card.md`：内部流程自动化的硬门槛
- `failure-modes.md`：常见失败模式
- `reusable-rules.md`：可复用判断规则

## 这组内容要保留的重点

- 业务场景
- 工程链路
- fallback
- 审计点
- 失败原因
- 复用规则

## 使用方式

- 招聘初筛：直接拿规则卡和失败模式做打回条件
- 推荐报告：直接引用 case card 里的证据链
- 销售样板：用成功案例 + 失败对照说明边界
- 复盘沉淀：把 human_edits / reuse_score / roi_signal 写回数据库
