# Case Card: 内部流程自动化成功样例

- case_id: `aie-ifw-success-001`
- direction: `AI 应用工程师 / 内部流程自动化`
- result: `success`
- sample_quality: `high`

## 场景

采购、法务和运营需要把审批材料、字段整理、文档摘要和系统流转做成一条可审计的 AI 工作流。

## 业务问题

- 手工搬运字段，重复录入多
- 审批摘要经常来回修改
- 高风险字段没有固定确认点
- 流程出了问题后，没人知道哪一步坏了

## 工程链路

- intake：表单 / 工单 / 邮件进入统一入口
- parse：结构化抽取关键字段
- lookup：查询 ERP / OA / 审批系统的系统 of record
- generate：生成摘要、补全说明和下一步建议
- confirm：高风险字段进入人工确认
- writeback：通过 API 或工单流转回写
- audit：记录输入、模型版本、人工改动、最终决策
- monitor：监控失败率、延迟、回退率和人工接管率

## Fallback

- 规则模板优先回退
- 高风险字段必须人工确认后再提交
- 模型异常时切到固定摘要模板
- 系统接口失败时保留草稿，不自动提交

## 审计点

- 原始输入与解析结果
- 关键字段谁改过、为什么改
- 哪一步触发了人工确认
- 何时回写系统 of record
- 失败是否进入回滚或重试

## 样板口径的结果

> 这是可卖样板的示意口径，不是正式生产统计。

- 试点中审批摘要整理时间从约 8 分钟降到约 3 分钟
- 重复录入和来回确认减少
- 失败时可回退到规则模板，不会阻断业务流转
- 业务方愿意把它当成“可运营系统”，而不是一次性 demo

## 为什么它成立

- 业务场景清晰
- 工程链路完整
- fallback 清楚
- 审计可追踪
- 结果能量化
