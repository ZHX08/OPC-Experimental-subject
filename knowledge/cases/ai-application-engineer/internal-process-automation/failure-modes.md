# Failure Modes: 内部流程自动化

下面这些是最常见、也最容易让方案“看起来像样、实际上不能卖”的失败模式。

## 1. 只会生成内容，不会接流程
- 现象：能写摘要、能改写表单，但没有回写系统
- 为什么危险：客户最后买到的是文案助手，不是自动化系统
- 修复动作：补入口、回写、人工确认和审计链路

## 2. 没有 system of record
- 现象：流程里每一步都靠模型猜
- 为什么危险：字段错了没人知道以谁为准
- 修复动作：明确 ERP / OA / 审批系统里的权威数据源

## 3. 没有审计
- 现象：看不出谁改了什么、模型输出过什么
- 为什么危险：内部流程一旦出错，无法追责和复盘
- 修复动作：记录输入、输出、人工编辑和版本号

## 4. 没有 fallback
- 现象：模型出错就卡住
- 为什么危险：业务流转被阻断
- 修复动作：规则模板、草稿保留、人工接管三选一至少要有一个

## 5. 没有人工确认点
- 现象：高风险字段自动提交
- 为什么危险：审批、法务、采购场景容错低
- 修复动作：把高风险字段拦在提交前

## 6. 没有监控 / 回滚
- 现象：上线后只能“祈祷模型正常”
- 为什么危险：任何异常都会放大成业务事故
- 修复动作：加监控、告警、灰度和回滚路径

## 7. 没有评测
- 现象：只展示 demo，没有可重复的验证
- 为什么危险：没人知道是不是比人工更好
- 修复动作：补抽样评测、人工复核和失败样本

## 8. 没有 ROI 口径
- 现象：只说“提效”，说不出省了多少时间、减少了多少错误
- 为什么危险：卖不动，也不好优先级排序
- 修复动作：把 ROI 绑定到时长、错误率、接单量或人工成本
