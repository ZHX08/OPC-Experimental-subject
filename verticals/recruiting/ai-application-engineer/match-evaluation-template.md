# AI 应用工程师匹配评估模板

> 这个模板只看一件事：候选人是不是能把 AI 应用做成“可上线、可评测、可迭代”的工程能力。

## 1. 评估结论
- 推荐级别：`强推 / 可推 / 谨慎推荐 / 不推荐`
- 总分：`/100`
- 是否满足必须项：`是 / 否`
- 结论一句话：`[一句话结论]`

## 2. 评分维度
建议按以下权重打分：
- 业务理解与场景拆解：20
- LLM 应用工程能力：25
- RAG / Prompt / Workflow 设计：20
- 评测 / 反馈 / 稳定性 / 成本控制：20
- 沟通协作与表达：15

## 3. 必须项核对
- 是否做过真实 AI 应用：`是 / 否`
- 是否能说明上线或准上线状态：`是 / 否`
- 是否具备工程实现能力：`是 / 否`
- 是否理解 fallback / 日志 / 评测：`是 / 否`
- 是否能把业务问题拆解成流程：`是 / 否`

## 4. 匹配亮点
- 亮点 1：`[例如：真实上线经验]`
- 亮点 2：`[例如：会做结构化输出和反馈回流]`
- 亮点 3：`[例如：能自己排查效果不稳定原因]`

## 5. 风险项
- 风险 1：`[例如：只做过 demo，没有长期线上运行证据]`
- 风险 2：`[例如：对成本、延迟、兜底考虑不足]`
- 风险 3：`[例如：技术栈很多，但无法说明自己实际负责的部分]`

## 6. 需要追问的问题
- 你负责的 AI 应用有没有上线？多少用户在用？
- 你怎么做评测？用什么指标判断结果“可用”？
- 你怎么做 RAG 的 chunking、rerank 和 prompt 调整？
- 出现幻觉、检索不到、调用失败时怎么兜底？
- 你如何平衡效果、成本和响应速度？

## 7. 示例评估
### 评估对象
基于 `resume-template.md` 中的“企业知识助手”与“销售方案生成助手”样例。

### 打分
- 业务理解与场景拆解：16 / 20
- LLM 应用工程能力：21 / 25
- RAG / Prompt / Workflow 设计：17 / 20
- 评测 / 反馈 / 稳定性 / 成本控制：15 / 20
- 沟通协作与表达：11 / 15
- **总分：80 / 100**

### 结论
- 推荐级别：可推
- 是否满足必须项：是
- 一句话结论：候选人具备 AI 应用工程落地经验，能做流程、能做工程、也知道怎么回收反馈；还需要进一步确认线上规模、评测方法和成本控制细节。

### 追问重点
- 是否有真实用户规模与稳定运行周期
- 是否能说清楚自己的贡献边界
- 是否有量化评测与改版记录
