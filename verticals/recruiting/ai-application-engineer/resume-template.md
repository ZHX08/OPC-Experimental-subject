## 2. 求职摘要

- 我做过什么类型的 AI 应用

### 示例
- 有 2 年 LLM 应用工程经验，做过企业知识助手、销售总结助手和客服辅助生成。

- 能把“能演示”的原型推进到“能稳定用”的上线版本。

## 4. 代表项目
> 建议每个项目都写清楚：业务场景、你的职责、技术栈、上线状态、结果指标。

### 项目名称：企业知识助手
- 场景：销售、运营和客服需要快速查内部知识
- 我的职责：
  - 负责检索链路设计、Prompt 设计和回答生成
  - 接入文档切分、向量检索和 rerank
  - 补齐日志、纠错回流和人工审核入口
- 技术栈：`Python / 向量检索 / RAG / rerank / 后端接口`
- 结果：
  - 检索命中率和答案可用性提升
  - 常见问题可直接自助解决，减少人工检索时间
- 可验证证据：`[Demo 链接 / 仓库 / 截图 / 评测记录]`

### 项目名称：销售方案生成助手
- 场景：根据客户需求自动整理方案初稿和跟进摘要
- 我的职责：
  - 设计结构化输入输出
  - 让生成结果能落到固定模板
  - 对高风险内容增加人工确认点
- 技术栈：`Prompt / workflow / JSON 输出 / 文档生成`
- 结果：
  - 方案初稿撰写时间缩短
  - 人工修改后可稳定复用
- 可验证证据：`[文档样例 / 结果截图 / 线上指标]`
