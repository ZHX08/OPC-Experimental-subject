# AI 应用工程师最小闭环（Step 1）

目标：先把一个岗位跑通，形成 `JD -> 简历 -> 匹配评估 -> 推荐报告 -> 反馈回流` 的最小可用闭环草案。

边界：
- 只覆盖 AI 应用工程师
- 不做 schema 重构
- 不扩展到其他岗位
- 不做自动化工作流升级

文件索引：
- `jd-template.md`
- `resume-template.md`
- `match-evaluation-template.md`
- `../../../templates/reports/ai-application-engineer-recommendation-enhancements.md`
- `../../../feedback/reviews/ai-application-engineer/feedback-loop-sample.md`

使用顺序：
1. 先补 JD
2. 再用简历模板采集候选人信息
3. 用匹配评估模板做初筛
4. 生成推荐报告时参考补强建议
5. 把人工修改结果回流到反馈样例里
