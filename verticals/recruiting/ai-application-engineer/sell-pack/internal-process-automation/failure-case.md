# 失败对照：把摘要助手误卖成审批自动化

## 典型错误

把一个只能生成合同摘要的工具，包装成“内部流程自动化方案”。

## 表面上看起来不错

- 能生成一段摘要
- 能把文档改写得更正式
- 演示过程流畅

## 实际问题

- 没有接 ERP / OA / 审批链
- 没有审计和回滚
- 没有人工确认点
- 没有监控和告警
- 没有试点数据

## 为什么不能卖

- 买到的是文案能力，不是流程能力
- 一旦字段出错，无法追责
- 一旦接口失败，业务会卡住
- 一旦模型胡写，没人能兜住

## 结论

这个案例只能算 prompt / 文案样例，不能当成 AI 应用工程师的内部流程自动化证据。
