# 试点核心指标模板

> 用途：统一口径，避免“看起来有提升”但其实无法复核。

## 1) 指标总览

### ROI
- 定义：试点带来的价值相对试点成本的回报
- 公式：`(节省人工时间价值 + 减少返工价值 + 避免错误损失 - 试点成本) / 试点成本`
- 数据源：试点日志、工时记录、返工记录、损失估算
- 频率：试点结束后汇总
- 红线：没有基线时不计算正式 ROI

### 效率
- 定义：单位流程从输入到完成所需的时间变化
- 公式：`(基线平均时长 - 试点平均时长) / 基线平均时长`
- 数据源：运行日志、人工计时、系统时间戳
- 频率：按批次 / 按天
- 红线：不能只看单次最快值，要看均值和 P95

### 错误率
- 定义：输出需要返工、纠错或重提的比例
- 公式：`错误或返工数量 / 总处理数量`
- 数据源：复核记录、返工单、人工修改记录
- 频率：按批次
- 红线：必须区分“轻微修正”和“业务级错误”

### 人工介入率
- 定义：流程中需要人工确认、人工改写或人工接管的比例
- 公式：`人工介入次数 / 总处理数量`
- 数据源：人工确认记录、审批记录、修改记录
- 频率：按批次
- 红线：要区分高风险字段确认和低风险润色

### fallback 率
- 定义：自动化无法继续时切换到规则模板或人工路径的比例
- 公式：`fallback 触发次数 / 总处理数量`
- 数据源：fallback 日志、异常告警、工单
- 频率：按批次
- 红线：fallback 是健康机制，不是失败本身；关键是能否继续业务流转

### 审计覆盖率
- 定义：有完整留痕的流程比例
- 公式：`有完整审计记录的流程数量 / 总流程数量`
- 数据源：审计日志、版本记录、系统事件
- 频率：按批次
- 红线：关键字段、人工修改和最终提交结果必须能追溯

### 回写成功率
- 定义：成功写回 system of record 的比例
- 公式：`成功回写数量 / 总回写尝试数量`
- 数据源：OA / ERP / 工单系统结果
- 频率：按批次
- 红线：如果不能回写，只能算半自动，不算完整试点

## 4) 建议的最终汇总口径

- 业务价值：节省多少时间、减少多少返工、避免多少错误
- 运营价值：流程是否更稳、是否更好审计
- 扩展价值：是否可以扩到第二条流程、第二个系统
- 风险判断：是否已经证明可以进入生产加固
