# 试点中证据记录模板

> 用途：每次运行、每批运行、每个关键节点都记录下来。别只记结论，要记过程。

## 1) 运行批次信息

- 试点编号：
- 批次编号：
- 记录日期：
- 记录人：
- 输入来源：
- 输入数量：
- 涉及流程节点：
- 本批次是否进入人工确认：`是 / 否`

## 2) 每次运行记录

对于每次运行，至少记录下面这些字段：

- run_id：
- 输入摘要：
- AI 输出摘要：
- 自动化是否完成：`是 / 否`
- 是否进入人工确认：`是 / 否`
- 人工修改点：
- 人工修改原因：
- 是否触发 fallback：`是 / 否`
- fallback 使用了什么：
- 是否继续回写：`是 / 否`
- 最终结果：
- 审计日志位置：
- 截图 / 录屏位置：
- 相关工单 / 单据编号：

## 3) 过程指标

- 本批次总输入数：
- 自动处理成功数：
- 进入人工确认数：
- 人工介入数：
- fallback 触发数：
- 处理失败数：
- 处理平均时长：
- P95 时长：
- 平均人工修改次数：
- 高风险字段拦截次数：
- 审计记录完整率：

## 4) 异常记录

- 异常编号：
- 异常发生时间：
- 异常类型：`输入问题 / 模型问题 / 接口问题 / 权限问题 / 审计问题 / 其他`
- 影响范围：
- 业务是否被阻断：`是 / 否`
- 采取的 fallback：
- 恢复用时：
- 是否需要复盘：`是 / 否`

## 5) 证据附件清单

- 输入原件
- AI 输出原稿
- 人工修改痕迹
- 最终提交稿
- 系统回写结果
- 审计日志
- 监控截图或告警记录
- 相关沟通记录

## 6) 当天结论

- 今天最好的结果：
- 今天最大的风险：
- 今天最明显的效率提升点：
- 今天最明显的错误来源：
- 明天要调整的点：
- 是否继续下一批：`是 / 否`
