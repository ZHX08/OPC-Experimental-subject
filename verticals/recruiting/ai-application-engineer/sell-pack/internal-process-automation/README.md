# 内部流程自动化最小可卖包

这是一套只围绕 **AI 应用工程师 -> 内部流程自动化** 的最小可卖样板。

## 目标

让客户一眼看懂三件事：
- 这个方向到底能解决什么业务问题
- 为什么它是“AI 应用工程师”的能力，而不只是文案 / Prompt 能力
- 风险边界和兜底在哪里

## 包含内容

- `success-case.md`：成功案例，一页讲清楚
- `failure-case.md`：失败对照，一页讲清楚什么不能卖
- `roi.md`：ROI / 价值说明
- `solution-one-pager.md`：一页方案说明
- `risk-boundary.md`：一页风险边界与兜底说明

## 使用方式

1. 先用成功案例建立信任
2. 再用失败对照说明边界
3. 用 ROI 说明为什么值得做
4. 用一页方案说明怎么落地
5. 用风险边界说明哪些事必须人工兜底

## 结论

这套包已经足够当作“样板雏形”去讲，但仍然是**可继续补证据的初版**，不是最终商业版本。
