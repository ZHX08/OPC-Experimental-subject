内部流程自动化试点控制台

这是 AI 应用工程师 → 内部流程自动化 的第一版本地 UI。 目标很窄：把 试点输入 → 运行结果 → 人工确认 → 反馈回写 → 证据记录 串成一条能直接跑的闭环。

system of record: OA / ERP 人审点: 高风险字段确认 写回: 反馈回流包 + 证据归档 模式: shadow mode 起步
默认试点场景：采购申请预审与分流。
先做单流程、单入口、单 SoR、单人审点，避免一开始就做“万能工作流”。

本次试点怎么落地

  1. 填试点输入，尽量贴近真实单据。
  2. 点 运行试点评估，看缺字段、风险点和写回路径。
  3. 人工确认后，填回写包与证据。
  4. 点 保存试点证据，生成本地记录和反馈包。
